# 公众号选题评分细则（xiaoji-topic-curator）

本文件供 `xiaoji-topic-curator` 在打分与写报告时选读。第一版**不依赖**磁盘上的 `style.yaml` / `history.yaml`：账号偏好、黑名单、近期已写方向由**用户在对话中口头提供**；若用户未提供，按「小机学AI大模型」常见定位默认推断（AI、Agent、提示词、AI 编程、工具实测、流程拆解、行业解读；兼有个人成长/观点叙事），并在报告中注明「按默认定位推断，可补充纠正」。

## 三维度（每项 1～10 分）

### 热度（权重 30%）

- 多源简报中标记为「今日最重要」、或明显全网讨论高峰 → 8～10  
- 圈层内热、持续发酵 → 5～7  
- 长尾或弱时效 → 1～4  
- **多平台 / 多栏目重复出现** → 可在热度上 +1～2，封顶 10  

### 相关度（权重 40%）

与账号主线（AI 实操、工具、编程、Agent、提示词等）及用户当次声明的侧重点的契合程度：

- 直接命中主线 → 8～10  
- 间接相关（如政策、硬件、商业事件但可落到 AI/工作流视角）→ 5～7  
- 勉强挂钩 → 3～4  
- 基本无关 → 0～2，**通常不进入 Top 列表**  
- 用户声明的 **黑名单话题/措辞** → **该条淘汰**（相关度视为 0，不输出为推荐选题）

### 切入价值（权重 30%）

写成公众号长文是否「好看、有用」：

- 明显反直觉、信息差、可落地方法论 → 8～10  
- 有争议、可讨论、多视角 → 6～7  
- 纯资讯搬运、薄 → 3～4  
- 太大展不开或太浅无话 → 1～2  

### content_style 加成（可选，作用于切入价值，封顶 10）

若用户说明了偏好的内容风格，可对切入价值 +0～2（不超 10）：

| 风格   | 条件           | 加分 |
|--------|----------------|------|
| 干货   | 能写成教程/流程/SOP | +2 |
| 故事   | 有人物、转折      | +2 |
| 情绪   | 强共鸣或观点张力  | +2 |
| 热点   | 与当日头条强绑定  | +2 |
| 测评   | 产品/工具/方案对比 | +2 |

## 综合分

```
总分 = 热度×0.3 + 相关度×0.4 + min(10, 切入价值+风格加成)×0.3
```

**排序原则**：相关度优先于纯热度；总分接近时，优先保留相关度更高的条目。

## 近期重复（对话级）

若用户提到「最近写过」「上周刚发」等，对疑似同题 **扣 1～3 分**（自行判断重叠程度），并在该条下标注 `⚠️ 与用户口述近期内容可能重叠`。

## 常青选题（可选 2～3 条）

- 不依赖当日热搜；教程、方法论、工具清单、长期判断等。  
- 热度分**固定记 5**，相关度与切入价值照常打；条目标注 `🌲 常青`。  
- 可与热点混排，按总分排序。

## 输出每条选题建议携带的字段

- 选题短标题（区别于素材原标题）  
- **公众号标题示例**（20～28 字）  
- 切入角度（1～2 句）  
- 热度 / 相关度 / 切入价值（各 x/10）与总分  
- 点击率潜力：高/中/低 + 一句理由  
- 推荐框架：痛点型 / 故事型 / 清单型 / 对比型 / 热点解读型 等  
- 推荐理由（1～2 句）  
- 若素材偏海外/英文源，可加一句 **本土化提醒**（补中文案例、读者语境）

若无外部 SEO 数据，**不要编造** SEO 分数；可写「SEO：待写稿时用关键词自然融入」或省略数值。

## 自动化可选附录

若用户或流水线需要机器可读结果，可在 Markdown 报告后附加 `TOPIC_JSON`（见主 `SKILL.md`）。
